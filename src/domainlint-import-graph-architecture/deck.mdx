## **The Rules...**

<Side>

<Appear>

📋 Use barrel files (`index.ts`) to expose feature APIs

</Appear>

<Appear>

📋 Use barrel files to import from other feature

</Appear>

<Appear>

📋 No cyclic dependencies between features

</Appear>

<Appear>

📋 Features should only import from other features

</Appear>

</Side>

---

layout: sidedCode

---

### **Rule 1 & 2: Expose APIs via Barrel Files**

Use `index.ts` to expose feature APIs

```tsx
// features/ui/index.ts
export { Button } from "./components/Button";
export { Input } from "./components/Input";

// ❌ BAD - Importing internal files
import { Button } from "@/features/ui/components/Button";

// ✅ GOOD - Importing from barrel
import { Button } from "@/features/ui";
```

<Side>

<Appear>

**Why?**

</Appear>

<Appear>

- Enforces public API boundaries

</Appear>

<Appear>

- Clear contracts between features

</Appear>

<Appear>

- Easier to move to packages later

</Appear>

</Side>

---

layout: sidedCode

---

### **Rule 3: No Cyclic Dependencies**

Features shouldn't depend on each other cyclically

```tsx
// ❌ BAD - Cycle detected
// features/auth/index.ts
import { UserProfile } from "@/features/user-profile";

// features/user-profile/index.ts
import { logout } from "@/features/auth";
```

<Side>

<Appear>

**Why?**

</Appear>

<Appear>

- Makes code impossible to understand

</Appear>

<Appear>

- Blocks incremental refactoring

</Appear>

<Appear>

- Prevents module extraction

</Appear>

</Side>

---

layout: sidedCode

---

### **Rule 4: Features Isolation**

Features should only import from other features (not internals)

```tsx
// ❌ BAD - Direct internal import
import { getUserData } from "@/lib/api";

// ✅ GOOD - Import from feature API
import { getUserData } from "@/features/user-profile";

// ✅ BETTER - Share via common
import { getUserData } from "@/features/common";
```

<Side>

<Appear>

**Why?**

</Appear>

<Appear>

- Enables independent deployment

</Appear>

<Appear>

- Reduces blast radius of changes

</Appear>

<Appear>

- Clarifies dependencies

</Appear>

</Side>
