# Cognitive Load Theory

<Appear>

**Intrinsic** → the real complexity of the problem

</Appear>

<Appear>

**Extraneous** → tooling noise, incidental complexity

</Appear>

<Appear>

**Germane** → the effort that builds understanding

</Appear>

<Notes>

From cognitive science. The goal is to minimize extraneous load so you can maximize germane load — actual learning and understanding.

</Notes>

# The Idea

### "What if we could enforce good architecture?"

<Notes>

After seeing the same problems in project after project, I built a tool.

</Notes>

---
layout: centered

# domainlint

<Appear>

Builds a **dependency graph** of your codebase

</Appear>

<Appear>

Runs **rules** against this graph

</Appear>

<Appear>

- No cyclic dependencies
- Feature-based directories
- No deep imports — use barrel files
- No cross-feature cycles
- No imports from global scope

</Appear>

<Appear>

Supports **custom rules**

</Appear>

<Appear>

### "Architecture as code"

</Appear>

<Notes>

domainlint doesn't just lint files — it understands your architecture as a graph and enforces constraints on it.

</Notes>
