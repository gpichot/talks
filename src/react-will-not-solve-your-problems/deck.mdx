### **The DOM problem**

```ts
// ❌ Checking visibility triggers style recalc
const isVisible = el.style.display !== "none";

// ❌ Reading layout triggers reflow
const width = el.offsetWidth;
el.style.width = width + 10 + "px";

// ❌ Loop = layout thrashing
items.forEach((item) => {
  const h = item.offsetHeight; // read → reflow
  item.style.height = h + 5 + "px"; // write → reflow
});
```

<Side>

<Appear>

Every **read** after a **write** forces the browser to recalculate layout

</Appear>

<Appear>

In a loop? **Layout thrashing**.

</Appear>

<Appear>

Multiple components updating? **Repaint storms, UI freezes**

</Appear>

</Side>

<Notes>

Nearly every reading or writing DOM/CSS operation triggers layout recomputing. Things get slow very fast. It's fine for one element, but when multiple parts of the UI update at once, it compounds.

</Notes>
